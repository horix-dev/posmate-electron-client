# Labels

Recommended labels to create in the repository for triage and planning.

## Type
- `type:feature`
- `type:bug`
- `type:tech-debt`
- `type:epic`

## Priority
- `priority:urgent`
- `priority:high`
- `priority:medium`
- `priority:low`

## Status
- `status:blocked`
- `status:in-progress`
- `status:review`
- `status:ready-for-qa`
- `status:done`

## Area
- `area:pos`
- `area:products`
- `area:sales`
- `area:purchases`
- `area:sync`
- `area:electron`
- `area:api`

## Platform
- `platform:windows`
- `platform:mac`
- `platform:linux`
- `platform:web`

## Notes
- Keep labels lightweight; avoid duplicates.
- Use `type:*` + `area:*` + `priority:*` + `status:*` on each issue.
